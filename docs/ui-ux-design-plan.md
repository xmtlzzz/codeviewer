# CodeViewer UI/UX 设计 Plan

> **For agentic workers:** 本文档定义 CodeViewer 前端原型的整体 UI/UX 设计规范。所有原型页面必须严格遵循此文档。

**Goal:** 基于线框图创建简约、低 AI 感的代码统计仪表盘原型，支持明暗模式切换，交互逻辑统一，页面切换返回路径清晰。

**Design Principle:** 极简主义 — 去掉一切不必要的装饰，让数据本身成为主角。参考 GitHub Insights、Linear、Vercel Dashboard 的视觉语言：白底、细线、克制用色、系统字体。

---

## 1. 设计系统 (Design System)

### 1.1 色彩体系

采用 CSS 变量驱动明暗双模式。浅色为默认模式，深色为可切换模式。

| Token | 浅色模式 | 深色模式 | 用途 |
|-------|---------|---------|------|
| `--bg-primary` | `#FAFAFA` | `#0D0D0D` | 页面背景 |
| `--bg-card` | `#FFFFFF` | `#1A1A1A` | 卡片背景 |
| `--bg-hover` | `#F5F5F5` | `#242424` | 悬停态背景 |
| `--border` | `#E8E8E8` | `#2A2A2A` | 分隔线、卡片边框 |
| `--text-primary` | `#1A1A1A` | `#E5E5E5` | 主文本 |
| `--text-secondary` | `#666666` | `#999999` | 次要文本、标签 |
| `--text-tertiary` | `#999999` | `#666666` | 辅助信息、时间戳 |
| `--accent` | `#2563EB` | `#3B82F6` | 主强调色（链接、按钮） |
| `--positive` | `#059669` | `#10B981` | 新增行数 |
| `--negative` | `#DC2626` | `#EF4444` | 删除行数 |
| `--divider` | `#F0F0F0` | `#222222` | 列表内分隔线 |

**用色原则：**
- 页面 90% 面积为中性色（黑/白/灰），强调色仅用于关键数据和交互元素
- 不使用渐变背景
- 不使用半透明/毛玻璃效果
- 正负数据色仅用于数字本身，不用于整块背景

### 1.2 字体系统

```
--font-sans: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
             "Microsoft YaHei", sans-serif;
--font-mono: "SF Mono", "JetBrains Mono", "Cascadia Code", "Consolas", monospace;
```

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| Display | 32px | 600 | 1.3 | 总代码行数大数字 |
| H1 | 20px | 600 | 1.4 | 页面标题 |
| H2 | 16px | 600 | 1.4 | 卡片标题 |
| Body | 14px | 400 | 1.6 | 正文、列表内容 |
| Caption | 12px | 400 | 1.5 | 标签、时间戳、辅助说明 |
| Mono-Num | 14px | 500 | 1.4 | 数字统计值（等宽对齐） |

**原则：** 全部使用系统字体栈，不引入 Google Fonts 等外部字体。数字使用等宽字体保证对齐。

### 1.3 间距与圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-xs` | 4px | 图标与文字间距 |
| `--space-sm` | 8px | 卡片内元素间距 |
| `--space-md` | 12px | 卡片间距 |
| `--space-lg` | 16px | 卡片内边距 |
| `--space-xl` | 24px | 区块间距 |
| `--radius` | 6px | 卡片圆角 |
| `--radius-sm` | 4px | 按钮圆角 |

**原则：** 圆角统一 6px，不使用大圆角（12px+），保持工具感而非消费产品感。

### 1.4 图标

使用 Lucide Icons（SVG 内联），统一 20px 尺寸，`stroke-width: 1.5`。

需要的图标：
- `arrow-left` — 返回按钮
- `settings` — 设置入口
- `sun` / `moon` — 明暗模式切换
- `chevron-right` — 语言卡片跳转指示
- `git-branch` — 提交数图标
- `file-code` — 变更文件数图标
- `plus` / `minus` — 新增/删除行数
- `refresh` — 同步/刷新

**禁止：** 不使用 emoji 作为 UI 图标。

### 1.5 动效

| 场景 | 时长 | 缓动函数 |
|------|------|---------|
| 页面切换 | 200ms | `ease-out` |
| 悬停反馈 | 150ms | `ease` |
| 主题切换 | 150ms | `ease` |
| 数字加载 | 300ms | `ease-out` |

**原则：** 所有动效仅用 `opacity` 和 `transform`，不操作 `width/height`。尊重 `prefers-reduced-motion`。

### 2.2 导航规则

| 起点 | 终点 | 触发方式 | 返回方式 |
|------|------|---------|---------|
| Dashboard | Language Detail | 点击语言卡片 | 左上角返回按钮 / 浏览器后退 |
| Dashboard | Settings | 点击右上角设置图标 | 左上角返回按钮 / 浏览器后退 |
| Language Detail | Dashboard | 返回按钮 | — |
| Settings | Dashboard | 返回按钮 | — |

**统一交互规则：**
1. 所有二级页面左上角固定显示 `← 返回` 按钮，点击返回 Dashboard
2. Dashboard 右上角固定显示设置图标 + 明暗模式切换图标
3. 页面切换使用 `translateX` 滑动动效（前进向左滑，返回向右滑）
4. 返回按钮始终位于同一位置（左上角），不因页面内容变化而移位
5. 键盘 `Esc` 键等效于点击返回按钮

**语言卡片设计：**
- 整卡可点击，`cursor: pointer`
- 悬停：背景变为 `--bg-hover`，右侧 `chevron-right` 微移 2px
- 卡片内三列数据：变更文件 / 变更行数 / 变更日期，使用等宽字体
- 变更行数带 `+` / `-` 前缀，分别用 `--positive` / `--negative` 着色
- 卡片间用 1px `--divider` 分隔，无圆角间隔

**总计区域：**
- 大数字使用 `--font-mono`，32px，600 字重
- 不使用动画计数器（避免 AI 感），直接显示最终值
- 辅助信息用 `--text-tertiary`

## 4. 交互逻辑统一规范

### 4.1 Header 组件（所有页面共享）

```
┌─────────────────────────────────────────┐
│  [← 返回(二级页)]   [标题]    [☀/☾][⚙]  │
└─────────────────────────────────────────┘
```

**规则：**
- Header 固定在顶部，高度 56px，背景为 `--bg-card`，底部 1px `--border`
- Dashboard 页：左侧不显示返回按钮（首页），右侧显示主题切换 + 设置图标
- 二级页面：左侧显示返回按钮，右侧仅显示主题切换
- 标题居中显示，使用 H1 字号

### 4.2 页面切换动效

```css
/* 页面容器 */
.page {
  transition: transform 200ms ease-out, opacity 200ms ease-out;
}
/* 进入：从右侧滑入 */
.page-enter {
  transform: translateX(100%);
  opacity: 0;
}
.page-enter-active {
  transform: translateX(0);
  opacity: 1;
}
/* 返回：向右滑出 */
.page-exit {
  transform: translateX(-100%);
  opacity: 0;
}
```

**规则：**
- 前进（Dashboard → 子页面）：新页面从右侧滑入
- 返回（子页面 → Dashboard）：当前页面向右滑出
- 同时支持 `prefers-reduced-motion: reduce` 时禁用滑动，仅用 opacity 淡入淡出

### 4.3 空状态处理

所有页面在无数据时显示统一的空状态：
- 居中显示一个简洁的 SVG 插画（非 emoji）
- 一行说明文字（`--text-secondary`）
- 一个操作按钮（如 "添加仓库" / "绑定 GitHub"）

### 4.4 加载状态

- 不使用全屏 spinner
- 使用骨架屏（skeleton）：灰色块占位，带 `opacity` 脉冲动画
- 骨架块背景为 `--bg-hover`，圆角 `--radius-sm`

## 5. 明暗模式实现

### 5.1 切换机制

```html
<!-- HTML 根元素 -->
<html data-theme="light">
```

```css
/* CSS 变量定义 */
:root[data-theme="light"] {
  --bg-primary: #FAFAFA;
  --bg-card: #FFFFFF;
  /* ... 浅色值 */
}
:root[data-theme="dark"] {
  --bg-primary: #0D0D0D;
  --bg-card: #1A1A1A;
  /* ... 深色值 */
}
```

### 5.2 切换逻辑

```javascript
// 主题切换函数
function toggleTheme() {
  const html = document.documentElement;
  const current = html.getAttribute('data-theme');
  const next = current === 'light' ? 'dark' : 'light';
  html.setAttribute('data-theme', next);
  localStorage.setItem('codeviewer-theme', next);
}

// 初始化：localStorage > 系统偏好 > 默认浅色
function initTheme() {
  const saved = localStorage.getItem('codeviewer-theme');
  const systemDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
  const theme = saved || (systemDark ? 'dark' : 'light');
  document.documentElement.setAttribute('data-theme', theme);
}
```

**注：** 原型阶段使用 localStorage 模拟持久化。Tauri 应用中改为 Rust 侧配置读写。

### 5.3 切换 UI

- Dashboard 和 Settings 页面右上角显示主题切换图标
- 浅色模式显示月亮图标（点击切到深色），深色模式显示太阳图标
- 图标 200ms 旋转过渡

## 8. 反 AI 感设计清单

以下为减少 "AI 生成感" 的具体措施：

| 要避免的 | 替代方案 |
|---------|---------|
| 渐变背景（蓝紫渐变等） | 纯色背景 |
| 大圆角卡片（16px+） | 6px 统一小圆角 |
| 毛玻璃/半透明效果 | 实色卡片 |
| 过多强调色 | 90% 中性色 + 仅数据着色 |
| 居中 hero 大标题 | 左对齐信息层级 |
